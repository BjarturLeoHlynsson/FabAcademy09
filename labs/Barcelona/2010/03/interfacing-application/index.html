This week’s  assignment was to write an application to write a user interface to an input or Output Device from the desktop Computer.
This week, beyond trying to get the most out of the Assembly programming, I have been trying to work on the Python programming too.
The first think I checked was if I have understood the codes that Neil already made, and I was able to change some features of the input devices (for example hello.light.45.py) and I have seen in the code that the following points are good to notice..
Coordinate systems

The Python GUI takes the topLeft end of the Window as the origin and counts to the right for +X and Down for +Y, this is important in Writing equations Events, for example (taken from the hello.Light.45.py) Notice the following equation:

In this red-bounded line ‘x’ which is value to be read from the application, is represented in the original form of

X= X0 + Dx*Value

X0, being the the X value where the visualization begins (zero of the visualization ) but in this case, the visualization begins at 20% of the X- side of the Window and goes to 90 % of the X- side of the Window

So if the visualization is full, as in the case where the light intensity reaches maximum (or reverse?), it covers 70% of the X-side of window…  nice ha? And the if you see the lines of the code in the green box,

Canvas.coords(‘rect1’ , .2*WINDOW, .05*WINDOW, X, .2*WINDOW, )  and

Canvas.coords(‘rect2’ , X,  .05*WINDOW, .9*WINDOW, .2*WINDOW, )

This comes from  to point description of a rectangle which is putting only the points of the diagonal

RECTANGLE :  (X0,Y0, X1,Y1) being (XO,YO) the coordinate of the one end of the diagonal of the rectangle and (X1,Y1) the other end coordinate.

So, here the diagonal starts (for rect1 for example) at 20% of the WINDOW in X (Left to Right), 5% of the WINDOW inY (Top down) and the growth of the bargraph is expressed in Variable X Keeping the Y value of any rectangle formed  at 20% of the X-side of the WINDOW!!

So, we can define any point in the window INTERMS OF THE WINDOW, by taking the reference point  0,0

So, Keeping the idea of the representation, if I want to change the LIGHT SENSOR VALUE to be shown in the vertical direction, I have to express the vertical rectangle in the way I mentioned above…

I want the graph to start from top to from 20% of Window in X and 0 of Window in Y,  and all I needed to change is the rectangle description,  and yes I got is as shown here…

As you can see, I have changed and practiced with equations and values… you can compare what I changed with the commented lines.

And the code is found below… Notice how the variables change both to define the Rectangle to be filled and the Value line (now Y instead of X)

import serial

eps = 0.1 # filter time constant
filter = 0.0 # filtered value

def idle(parent,canvas):
global filter, eps
#
# idle routine

byte2 = 0
byte3 = 0
byte4 = 0
ser.flush()
while 1:
#
# find framing
#
byte1 = byte2
byte2 = byte3
byte3 = byte4

if ((byte1 == 1) & (byte2 == 2) & (byte3 == 3) & (byte4 == 4)):
break
low = ord(ser.read())
high = ord(ser.read())
value = 256*high + low
filter = (1-eps)*filter + eps*value
# x = int(.2*WINDOW + (.9-.2)*WINDOW*filter/1024.0)
#y = int(.7*WINDOW*filter/1024.0)
y = .7*WINDOW-int(.7*WINDOW*filter/1024.0)
canvas.itemconfigure(“text”,text=”%.1f”%filter)
#canvas.coords(‘rect1′,.2*WINDOW,.05*WINDOW,x,.2*WINDOW)
canvas.coords(‘rect1′,.2*WINDOW,.0,.4*WINDOW,y)
#canvas.coords(‘rect2′,x,.05*WINDOW,.9*WINDOW,.2*WINDOW)
canvas.coords(‘rect2′,.2*WINDOW,y,.4*WINDOW,.7*WINDOW)
canvas.update()
parent.after_idle(idle,parent,canvas)

print “command line: hello.light.45.py serial_port”

canvas.create_text(.1*WINDOW,.125*WINDOW,text=”Value”,font=(“Helvetica”, 24),tags=”text”,fill=”#0000b0″)
#canvas.create_rectangle(.2*WINDOW,.05*WINDOW,.3*WINDOW,.2*WINDOW, tags=’rect1′, fill=’#b00000′)

# start idle loop
#

root.mainloop()
