Machine Design: Mantis 9

Once you receive the box with all the components, everyone will feel different things. One person will feel happy, one person might wonder what all the items are for, and others might feel nervous feeling it will take a lot of time to assemble it. Whatever the mood is, you realize this is a team work that requires patience, precision, a lot of organization, cooperation, and a clean table where you can put all the machine parts. Mantis 9 has its instructions included and a listÂ  of the items. Basically, this work is divided into the mechanical and electrical part. All that information will be found in the Internet.

For the assembling, there are two groups of videos. Part 1 is about the assembly of X, Y and Z axes, and Part 2 is about motor/leadscrew installation and final assembly. Everything went fine; however, we don’t have the spindle yet. The spindle shaft should have a hole, but it has not been done yet. To sum up, you will not have so much trouble with the parts since everything was perfectly explained in the videos…You will have to be careful when using the Loctite, and the epoxy, not only because of the smell, but also because you won’t want to stick all the parts!

About the electronics, we began with the Fabnet Adaptor Board. For each board, you will find the OVERVIEW where you have explained what it is for, the MAKE IT showing you where the components are, FILES, and BOM with the list of the components for making that boad. In FILES, you will a PNG with the drawing to be machined. It was our first one, and we machined more than we were supposed to. That’s why we used a white cable to join the paths.

It is important that you read all the instructions properly. We say that because we machined two boards: the low and higher currend driver boards. Actually, we didn’t have to make them. You realize that when you see that you don’t have all the components to finish them. Fortunately, Internet exists, and by sending an email we found out that the boards we are using are the D.Carr Driver Boards.

These are the D.Carr Driver Boards A3982-1.0. The quantity is 3, and according to the link, the board works with a parallel interface. We have one resistor left, but it depends on the current. There is one value for 1A and another one for 2A. We still have to measure it.

The following item is the H-Bridge (076-004). You will need one of this. In this case, you have to set the fuse to 0×76.

Finally, we haveÂ  the Stepper Motor Controller (076-001). We had some trouble when tryning to set the fuses. For this case, you will need to set to 0xDE. At the beginning, the area near tha arrow got a little bit hot. When soldering the 120 ohm resistor, it was better; moreover, we were able to set the fuse.Â  Then, in order to program it, go to FILES and choose 076-001e.asm, which is Firmware Rev.E: Motion Queue.

Embedded networking and communications

This time the assignment was about networking and communications. The idea is to create an IP thanks to some PCBs and using that number we would be able to get the communication between the computer and the boards.

The first important thing is to download the .cad files: one hub, one serial and two nodes. For the last two PCBs, be careful when soldering the LEDs and diodes since they have polarity. Do not forget that the line shows whichÂ  the cathode is. When programming, initialize the 20 MHz crystals as it appears in the image, and then, use the .hex for each one.

The following step is to connect all the PCBs, including the serial cable and the 9V battery, and then, run the python program. As I explained, you will use either ttyS0 or ttyS1. The .py file is just as it appears on top.

After the .py file, 3 things will occur as I explained. The LEDs will be turned on, but when pressing one of the buttons you will be able to control the other LED (I mean….you will turn it on and off). When doing it, an IP will be generated, and a window (thtpv4.py) will open showing you that IP.

Finally, after getting your IP, you have to type it in the browser like it is shown in 1. After this, you will be able to see the state of the LED in the screen. You will only have to push the button and refresh. The messages are ‘light is on‘ and ‘light is off‘. I suppose you can also control the LED from the computer, but I think it requires some programming I still have to practice. In conclusion, the goals of learning this is to know we have the chance to understand about networking and communication. We can have the option to control something from a different location, and more.
